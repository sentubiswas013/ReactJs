**Best Coding Practices in Java**

### 1. Follow Naming Conventions

Use meaningful and consistent names.

```java
class EmployeeService {}
int employeeCount;
void calculateSalary() {}
```

**Benefits:** Better readability and maintainability.

### 2. Keep Methods Small and Focused

A method should do **one thing only**.

```java
public void processOrder(Order order) {
    validateOrder(order);
    saveOrder(order);
    sendNotification(order);
}
```

### 3. Prefer Constructor Injection

Make dependencies explicit and immutable.

```java
@Service
public class UserService {
    private final UserRepository repository;

    public UserService(UserRepository repository) {
        this.repository = repository;
    }
}
```

### 4. Program to Interfaces

Depend on abstractions, not implementations.

```java
List<String> names = new ArrayList<>();
```

Instead of:

```java
ArrayList<String> names = new ArrayList<>();
```

### 5. Use `final` Wherever Possible

Prevents accidental modification.

```java
final String companyName = "OpenAI";
```

### 6. Handle Exceptions Properly

Catch only what you can handle.

```java
try {
    fileService.read();
} catch (IOException e) {
    log.error("File read failed", e);
}
```

Avoid:

```java
catch (Exception e) {}
```

### 7. Never Ignore Exceptions

Bad:

```java
catch (Exception e) {
}
```

Good:

```java
catch (Exception e) {
    log.error("Unexpected error", e);
}
```

### 8. Use `Optional` Carefully

Avoid returning `null`.

```java
public Optional<User> findUser(Long id) {
    return repository.findById(id);
}
```

### 9. Use Streams Judiciously

Use streams when they improve readability.

```java
List<String> names = users.stream()
        .map(User::getName)
        .toList();
```

Don't force streams for complex logic.

### 10. Override `equals()` and `hashCode()`

Required when objects are used in collections.

```java
@Override
public boolean equals(Object obj) { ... }

@Override
public int hashCode() { ... }
```

### 11. Use Enums Instead of Constants

```java
public enum OrderStatus {
    PENDING, APPROVED, REJECTED
}
```

Instead of:

```java
public static final String PENDING = "PENDING";
```

### 12. Avoid Magic Numbers

Bad:

```java
if (age > 18)
```

Good:

```java
private static final int ADULT_AGE = 18;

if (age > ADULT_AGE)
```

### 13. Write Immutable Classes When Possible

```java
public final class Employee {
    private final String name;

    public Employee(String name) {
        this.name = name;
    }

    public String getName() {
        return name;
    }
}
```

### 14. Use Logging Instead of `System.out.println`

```java
private static final Logger log =
        LoggerFactory.getLogger(UserService.class);

log.info("User created");
```

### 15. Close Resources Properly

```java
try (BufferedReader reader = new BufferedReader(new FileReader(file))) {
    return reader.readLine();
}
```

### 16. Avoid Code Duplication (DRY)

Bad:

```java
calculateTaxForEmployee();
calculateTaxForManager();
```

Good:

```java
calculateTax(Person person);
```

### 17. Write Unit Tests

```java
@Test
void shouldCalculateDiscount() {
    assertEquals(10, service.getDiscount(100));
}
```

### 18. Use Constants for Repeated Values

```java
private static final String ACTIVE = "ACTIVE";
```

### 19. Keep Classes Cohesive

A class should have **one responsibility**.

```java
UserService     // business logic
UserRepository  // database access
```

### 20. Follow SOLID Principles

* **S** – Single Responsibility Principle
* **O** – Open/Closed Principle
* **L** – Liskov Substitution Principle
* **I** – Interface Segregation Principle
* **D** – Dependency Inversion Principle

### Most Important Interview-Worthy Practices

1. **Meaningful naming**
2. **Small methods**
3. **Constructor injection**
4. **Exception handling**
5. **Immutability**
6. **SOLID principles**
7. **Program to interfaces**
8. **Avoid code duplication**
9. **Proper logging**
10. **Unit testing**

These practices make code **readable**, **maintainable**, **testable**, **scalable**, and **production-ready**.
